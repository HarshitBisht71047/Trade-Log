# 📈 TradeLog

**TradeLog** is a modern, privacy-focused, and highly visual trading journal designed to help day traders and investors find their edge. Built with performance and aesthetics in mind, it provides seamless trade tracking, emotional analysis, and comprehensive data visualizations directly in your browser.

🚀 **[View Live Application](https://trade-journal-app-puce.vercel.app)**

## ✨ Features

- **Offline-First & Secure:** All of your trading data, passwords (hashed via SHA-256), and custom settings are stored entirely locally in your browser. No external databases means your edge remains your secret.
- **Advanced Analytics Dashboard:** Automatically track total P&L, Daily ROI, Win Rate, and Average Trades Per Day with interactive donut and bar charts.
- **Emotional Heatmap Analysis:** Visualize exactly which emotional states (e.g., FOMO, Revenge Trading, Patience) correlate to your biggest wins and losses.
- **Smart Broker Import (Beta):** Instantly upload CSV or Excel (`.xlsx`, `.xls`) files exported directly from your broker. TradeLog's smart parser automatically maps the correct columns for you.
- **Rich Media Support:** Upload screenshots of your chart setups directly into your journal entries for visual review.
- **Customizable Environment:** Supports full Light/Dark mode toggling, multiple currencies, custom strategy tags, and configurable daily trade limits to visually warn you against overtrading.

## 🛠️ Technology Stack

- **Frontend Core:** Vanilla HTML, CSS, JavaScript
- **Framework:** React 18 (imported via CDN for lightweight execution)
- **Data Visualization:** Chart.js
- **Iconography:** Phosphor Icons
- **Data Parsing:** SheetJS (for complex broker Excel exports)
- **Deployment:** Vercel

## 🚀 Getting Started Locally

Because TradeLog requires zero backend configuration, running it locally is incredibly simple:

1. Clone the repository:
   ```bash
   git clone https://github.com/HarshitBisht71047/Trade-Log.git
   ```
2. Navigate into the project directory:
   ```bash
   cd Trade-Log
   ```
3. Open `index.html` in your web browser. 
   *(Note: For the best experience, consider using a simple local server like VS Code's "Live Server" extension).*
